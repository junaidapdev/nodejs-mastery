# Code-Along

> **Purpose:** Small snippets and experiments I typed during lectures. Not full projects — just the bits where I forced myself to type along instead of just watching.

---

## How I organize this

Loose folders by week or topic. Don't over-engineer the structure here — these are scratch files. Examples:

```
code-along/
├── week-1-event-emitter/
├── week-1-http-server/        ← my Saturday challenge
├── week-2-middleware-experiments/
├── week-3-drizzle-queries/
└── ...
```

Each folder gets a tiny `README.md` saying what I was experimenting with.

---

## What goes here vs in `projects/`

- **Here:** snippets, experiments, "what happens if I..." tests
- **`projects/`:** the two real builds (URL Shortener, Basecamp clone)
